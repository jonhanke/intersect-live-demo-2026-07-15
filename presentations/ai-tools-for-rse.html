Claude Code

What is Claude Code?

  • ⚡
    Agentic CLI toolRuns in your terminal. Reads files, runs commands, edits code — autonomously.
  • 🧠
    Whole-codebase awarenessUnderstands project structure, conventions, and dependencies. Not just the snippet you paste.
  • 🔧
    Real tool useRuns tests, linters, builds, git commands — then responds to what it sees.
  • 🔌
    IDE integrationVS Code, Cursor, and Windsurf extensions available. Run parallel instances in split panes.

$ npm install -g @anthropic-ai/claude-code

# Start in your project
$ cd my-research-project
$ claude

Claude Code — Key Feature

CLAUDE.md — Your Project's Brain

CLAUDE.md lives in your project root and is read at the start of every Claude Code session. Think of it as persistent working memory for your project.

  • 🏗️
    Build & test commandsnpm run test, make build, pytest — so Claude always runs the right thing
  • 📐
    Code style rules"Use ES modules, not CommonJS" · "Always type hint Python functions"
  • 📁
    Architecture pointers"State is managed in src/stores/" · "Data validation lives in validators/"
  • ⚠️
    Off-limits areasFiles or patterns Claude should never modify without asking
# CLAUDE.md

## Build Commands
- make test — run full test suite
- make lint — flake8 + mypy
- make docs — build Sphinx docs

## Code Style
- Python 3.11+, type hints required
- Numpy docstrings everywhere
- No globals — use config objects

## Architecture
- Data I/O → src/io/
- Analysis → src/analysis/
- Never edit data/raw/ directly

# Keep under ~150 lines for best results

Claude Code — Critical Concept

Context is Your Most Precious Resource

⚠️ Why Context Degrades

Every message, file read, and command output fills the context window. As it fills, Claude "forgets" earlier instructions and makes more mistakes. This is the primary failure mode.

✅ Context Hygiene

  • Use /clear often — start fresh for each new task
  • Use /compact to summarize long sessions
  • Reference files with @path rather than pasting contents
  • Keep MCPs lean — 20k+ tokens of MCPs cripples available working space
  • Monitor the context usage indicator continuously

💡 Pro Tips for Context

  • Break large tasks into smaller, independent sessions
  • Use CLAUDE.md so you don't have to repeat project context each session
  • Pipe targeted data: cat file | claude
  • For parallel work: separate git worktrees + separate Claude instances
  • Up arrow navigates back to past sessions — handy for yesterday's context
# Start fresh for each task
> /clear

# Summarize and continue
> /compact
Claude Code — Best Practices

Make Verification Rock-Solid

Claude can check its own work — but only if you give it clear signals. Build verification into every task.

🧪

Test Suites

Tell Claude to run your tests after every change. "Don't consider it done until all tests pass." Define the command in CLAUDE.md.

🖥️

Claude in Chrome

For UI work: the browser extension lets Claude open tabs, inspect DOM/console/network, and iterate until the UI matches expectations.

🔗

Hooks (Advanced)

Stop hooks run after each response. Auto-trigger builds, type checks, linters. If < 5 errors, show Claude. If ≥ 5, trigger a fix agent.

Include tests, screenshots, or expected outputs so Claude can check itself. Invest in making your verification rock-solid.
— Anthropic Claude Code docs
Claude Code — Power Features

Going Further: Parallel & Advanced

⚡⚡

Parallel Sessions with Git Worktrees

Run multiple Claude instances simultaneously on different features — each in its own isolated git worktree. Zero conflicts, full parallelism. Kick off a refactor in the cloud, go for coffee, then pull it back locally to review.

🤖

Sub-Agents & Skills

Create custom agents in .claude/agents/ with their own tools and permissions. Build feature-specific agents (e.g., "frontend-error-fixer", "build-error-resolver") instead of one overloaded agent.

🔌

MCP Servers

Model Context Protocol extends Claude with external tools — databases, GitHub, Slack, custom data sources. Keep MCP token usage lean (under 20k) to preserve working context.

🔔

Hooks & Notifications

Set up hooks to get a text message when a long-running agent finishes. Stop babysitting Claude — let it work and come back when it's done.

Speech-to-Text

Super Whisper

  • 🎙️
    OpenAI Whisper poweredRuns on-device or via API. Exceptional accuracy, especially for technical vocabulary.
  • ⌨️
    Global hotkeyWorks system-wide. Dictate into any app — terminal, editor, browser, Slack.
  • 🧬
    Custom vocabularyTeach it your domain terms: library names, file paths, technical jargon, colleague names.
  • ✍️
    Post-processing promptsAuto-format transcript as a Claude prompt, code comment, Git commit message, etc.
  • 💸
    ModesLocal (free, private) or cloud (faster, more accurate). Subscription model for macOS.

🏆 Best For

  • Dictating long, structured prompts to Claude Code
  • Writing commit messages and PR descriptions hands-free
  • Capturing ideas during code review without breaking flow
  • Drafting documentation or comments while reading code

💡 Top Tips

  • Add your project's key terms to custom vocabulary immediately
  • Create a "Claude prompt" output mode to format speech as structured commands
  • Use local mode for anything sensitive — no data leaves your machine
Speech-to-Text

macOS Built-in Dictation

  • 🍎
    Completely free & built-inSystem Settings → Keyboard → Dictation. Press Fn twice (or custom key) to activate.
  • 🔒
    On-device, privateEnhanced Dictation processes entirely locally. Nothing sent to Apple servers.
  • 🌐
    Works anywhereAny text input on macOS — Xcode, VS Code, Terminal, browsers, everything.
  • 🗣️
    Voice commandsSay "new line", "open bracket", "period", "all caps [word]" for hands-free formatting.

Power Combo

Voice + Claude Code = Flow State

Hands stay on keyboard for code review. Voice handles intent, context, and direction.

🎯 Example Workflow

1. Read through code silently
2. Hold hotkey → dictate your intent
"Add error handling for the case where the API returns null, log it with our standard logger, and write a test for it" 3. Super Whisper formats as a Claude prompt
4. Paste into Claude Code → let it run
5. Review changes while dictating next task

✅ What to Dictate vs. Type

Dictate:

  • High-level intent and requirements
  • Context and constraints for Claude
  • Commit messages, PR descriptions
  • Comments and documentation

Type:

  • Code, file paths, exact syntax
  • Slash commands (/clear, @path)
  • Anything with special characters
